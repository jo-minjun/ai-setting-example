# 일반 코딩 작업 프롬프트 가이드

## 프롬프트 구조

```markdown
# [작업 유형]: [작업 제목]

## 목표
[달성하려는 결과물]

## 맥락
- 프로젝트: [프로젝트 정보]
- 기술 스택: [사용 기술]
- 관련 파일: [작업 대상 파일들]

## 요구사항
1. [구체적 요구사항 1]
2. [구체적 요구사항 2]
...

## 제약사항
- [지켜야 할 규칙/제한]

## 참고 사항
- [기존 코드 패턴, 컨벤션 등]
```

## 작업 유형별 템플릿

### 기능 구현

```markdown
# 기능 구현: 사용자 인증 API

## 목표
JWT 기반 로그인/로그아웃 API 구현

## 맥락
- 프로젝트: Spring Boot 4.0 REST API
- 기술 스택: Java 21, Spring Security
- 관련 파일: src/main/java/.../auth/

## 요구사항
1. POST /api/auth/login - 로그인
2. POST /api/auth/logout - 로그아웃
3. GET /api/auth/me - 현재 사용자 정보

## 제약사항
- DDD 아키텍처 준수
- Google Java Format 적용

## 참고 사항
- 기존 UserRepository 활용
```

### 버그 수정

```markdown
# 버그 수정: 주문 금액 계산 오류

## 현상
할인 적용 시 총 금액이 음수로 표시됨

## 재현 조건
1. 상품 추가 (1000원)
2. 50% 할인 쿠폰 적용
3. 추가 할인 1000원 적용
→ 총액: -500원 (예상: 0원)

## 예상 원인
할인 금액 합계가 상품 금액을 초과하는 경우 처리 누락

## 관련 파일
- OrderService.java:calculateTotal()
- DiscountPolicy.java

## 기대 결과
총액은 최소 0원 이상이어야 함
```

### 리팩토링

```markdown
# 리팩토링: 결제 서비스 분리

## 목표
PaymentService의 책임 분리로 유지보수성 향상

## 현재 문제
- 단일 클래스에 600줄 이상
- 결제, 환불, 정산 로직 혼재
- 테스트 어려움

## 리팩토링 방향
1. PaymentService → 결제 처리
2. RefundService → 환불 처리
3. SettlementService → 정산 처리

## 제약사항
- 기존 API 인터페이스 유지
- 단계적 마이그레이션 가능하게
```

## 핵심 포인트

1. **목표 명확화**: 무엇을 달성해야 하는지 구체적으로
2. **맥락 제공**: 프로젝트 환경, 기존 코드 정보
3. **제약사항 명시**: 따라야 할 규칙, 컨벤션
4. **재현 가능성**: 버그는 재현 조건 필수
