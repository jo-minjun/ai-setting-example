# 에이전트 생성 프롬프트 가이드

## 프롬프트 구조

```markdown
# [에이전트명] 에이전트 생성 요청

## 역할
[에이전트가 수행할 전문 역할]

## 사용 시나리오
[언제 이 에이전트가 호출되어야 하는지]
- 예시 요청 1: "..."
- 예시 요청 2: "..."

## 필요 도구
[에이전트가 사용해야 할 도구들]
- Read: [용도]
- Bash: [용도]
- Grep: [용도]
...

## 작업 흐름
1. [단계 1]
2. [단계 2]
...

## 출력 형식
[에이전트가 반환해야 할 결과 형식]

## 제약사항
- [행동 제한, 규칙 등]
```

## 좋은 프롬프트 예시

```markdown
# 코드 리뷰어 에이전트 생성 요청

## 역할
Pull Request 또는 코드 변경사항을 분석하여 리뷰 의견 제공

## 사용 시나리오
- "이 PR 리뷰해줘"
- "코드 변경사항 검토해줘"
- "이 커밋 리뷰해줘"

## 필요 도구
- Bash: git diff, git log 실행
- Read: 변경된 파일 읽기
- Grep: 패턴 검색

## 작업 흐름
1. 변경된 파일 목록 확인 (git diff --name-only)
2. 각 파일의 변경 내용 분석
3. 코드 품질, 버그 가능성, 개선점 도출
4. 구조화된 리뷰 의견 작성

## 출력 형식
- 요약: 전체 변경사항 개요
- 이슈: 발견된 문제점 (심각도 표시)
- 제안: 개선 사항
- 칭찬: 좋은 부분

## 제약사항
- 파일 수정 금지 (읽기만)
- 리뷰 의견은 건설적으로 작성
```

## 핵심 포인트

1. **도구 명시**: 에이전트가 사용할 도구를 구체적으로 지정
2. **행동 제한**: 에이전트가 하면 안 되는 것 명확히 기술
3. **출력 형식**: 결과물 구조를 사전에 정의
