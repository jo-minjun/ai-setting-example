# 워크플로우 패턴

## 순차적 워크플로우

복잡한 작업의 경우, 작업을 명확하고 순차적인 단계로 나눈다. SKILL.md 초반에 Claude에게 프로세스 개요를 제공하면 종종 도움이 된다:

```markdown
PDF 폼 채우기는 다음 단계를 포함한다:

1. 폼 분석 (analyze_form.py 실행)
2. 필드 매핑 생성 (fields.json 편집)
3. 매핑 검증 (validate_fields.py 실행)
4. 폼 채우기 (fill_form.py 실행)
5. 출력 확인 (verify_output.py 실행)
```

## 조건부 워크플로우

분기 로직이 있는 작업의 경우, Claude를 의사결정 지점으로 안내한다:

```markdown
1. 수정 유형을 결정한다:
   **새 콘텐츠 생성?** → 아래 "생성 워크플로우" 따라가기
   **기존 콘텐츠 편집?** → 아래 "편집 워크플로우" 따라가기

2. 생성 워크플로우: [단계]
3. 편집 워크플로우: [단계]
```
